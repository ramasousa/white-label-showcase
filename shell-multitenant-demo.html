O artefato JavaScript é idêntico para os dois

Vivo e Claro recebem o mesmo bundle do Shell — mesma URL, mesmo hash de integridade. O que difere é o partnerId passado como parâmetro. Na inicialização, o Shell chama GET /v1/partners/{partnerId}/config e recebe os Design Tokens, produtos habilitados e UCT scope. Zero código condicional dentro do Shell para distinguir parceiros.

O Shell visto por dentro — mesmo código, dois contextos

Observe que a única diferença entre os dois fluxos é o valor de partnerId. O Shell não tem if (partner === 'vivo') em lugar nenhum.

Vivo · Shell boot 
// App do parceiro Vivo
<ShellApp
  partnerId="vivo"
  onEvent={handleEvent}
/>
