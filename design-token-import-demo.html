O mesmo MFE de Capital de Giro recebe os Design Tokens do Cacau Show e d'O Boticário. Observe em tempo real a injeção dos tokens CSS, a transformação visual e a inspeção dos valores aplicados.

⚡
O MFE não tem uma linha de cor hardcoded. Todos os valores visuais — cor primária, tipografia, border-radius, sombras — são CSS Custom Properties injetadas pelo Shell ao carregar o pacote de tokens do parceiro. O mesmo bundle JavaScript renderiza identidades completamente diferentes.

Como o Shell aplica os tokens no MFE
Shell · applyDesignTokens.ts 
async function applyDesignTokens(partnerId: string, mfeEl: HTMLElement) {

  // 1. Busca o pacote de tokens do parceiro via Partner Config API
  const config = await fetch(`/v1/partners/${partnerId}/design-tokens`);
  const tokens: DesignTokenPackage = await config.json();

  // 2. Sanitiza o pacote — evita injeção de CSS malicioso
  const safe = sanitizeTokens(tokens); // allowlist de propriedades permitidas

  // 3. Aplica como CSS Custom Properties no elemento raiz do MFE
  // Scoped ao MFE — não vaza para outros parceiros no mesmo Shell
  mfeEl.style.setProperty('--mfe-primary',   safe.color.primary);
  mfeEl.style.setProperty('--mfe-secondary', safe.color.secondary);
  mfeEl.style.setProperty('--mfe-accent',    safe.color.accent);
  mfeEl.style.setProperty('--mfe-surface',   safe.color.surface);
  mfeEl.style.setProperty('--mfe-radius',    safe.shape.radius);
  mfeEl.style.setProperty('--mfe-shadow',    safe.elevation.card);
  mfeEl.style.setProperty('--mfe-font',      safe.typography.family);

  // 4. Atualiza o logo do MFE com o nome do parceiro
  mfeEl.querySelector('.mfe-nav-logo').textContent = tokens.meta.appName;
}

// Os tokens vivem como variáveis CSS — o MFE usa var(--mfe-primary)
// em todos os lugares. Zero hardcode. Zero rebuild.
