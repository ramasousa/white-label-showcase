Sem Login vs Com Login

Compare os dois modelos de autenticação do portal white label. No modo Federado, o parceiro já autenticou o usuário. No modo Standalone, o Bradesco/Aarin é responsável pelo login — com MFA, segurança de credenciais e todos os riscos que isso implica.

🔗
Modo Federado
O parceiro autentica o usuário em seus próprios sistemas e repassa o token via Token Exchange. O Shell não exibe tela de login.
sem tela de login parceiro é o IdP menor risco regulatório
🔐
Modo Standalone
Bradesco/Aarin é o Identity Provider. Portal completo com login, senha, MFA e recuperação de conta. Maior responsabilidade de segurança.
login + MFA maior responsabilidade riscos de fraude

Modo Standalone — Responsabilidades Adicionais Ativas
O Bradesco/Aarin passa a ser o guardião da identidade. Isso ativa: MFA obrigatório para operações de crédito · proteção contra credential stuffing · logs de autenticação retidos por 5 anos (Bacen) · responsabilidade LGPD como controlador de dados · pentest específico do fluxo de login · monitoramento de phishing do domínio branco.
Comparação: Federado vs Standalone

Impacto em segurança, responsabilidade e complexidade de implementação.

Dimensão Federado (sem login) Standalone (com login)
Quem autentica o usuário Parceiro (IdP próprio) Bradesco/Aarin
Tela de login no portal Não existe — Shell já recebe UCT Sim — login + senha + MFA obrigatório
Risco de phishing Baixo — parceiro é alvo, não o portal Alto — portal branco é alvo direto
Credential stuffing Não se aplica Risco real — rate limit + CAPTCHA obrigatórios
SIM Swap / MFA bypass Não se aplica Risco se MFA via SMS — preferir TOTP
Responsabilidade LGPD Parceiro é controlador dos dados de auth Bradesco/Aarin é controlador — obrigações maiores
Compliance Bacen 4.658 MFA recomendado MFA obrigatório para operações financeiras
Retenção de logs de auth Compartilhada com parceiro 100% responsabilidade Bradesco · mín. 5 anos
Complexidade de integração Parceiro implementa Token Exchange Bradesco entrega portal pronto — menor esforço do parceiro
Parceiros sem infra tech Difícil — exige capacidade de Token Exchange Perfeito — parceiro só distribui o link
Recuperação de senha Não existe no portal BaaS Vetor crítico de fraude — exige validação rigorosa
